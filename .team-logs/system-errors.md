# 体系错误追踪

> 记录开发体系本身的缺陷和摩擦。积累 5 条以上未修复项时，提醒用户优化 CLAUDE.md。
> 这不是代码 bug 记录（那个在 bug-history.md），而是**流程/规则/协作方式**的问题。

## 格式

```
### SE-XXX: [一句话描述]
- **发现日期**: YYYY-MM-DD
- **类型**: 规则冲突 | 流程堵点 | Agent 行为偏差 | 规则缺失 | 规则过度
- **现象**: 具体发生了什么
- **根因**: 为什么会这样
- **影响**: 浪费了多少时间/造成了什么后果
- **状态**: 待修复 | 已纳入 CLAUDE.md | 已废弃
- **修复方案**: （如已修复，说明改了什么）
```
