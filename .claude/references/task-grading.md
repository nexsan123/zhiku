# 任务分级制度（铁律 TG-01）

> 从 CLAUDE.md 提取。Lead 判定级别时按需加载。

## 分级标准

| 级别 | 名称 | 判定标准 | 示例 |
|------|------|----------|------|
| **S** | 小修 | 单文件 ≤50 行，不涉及新接口/新页面 | Bug 修复、文案修改、样式微调 |
| **M** | 中改 | 单模块 ≤3 文件，不跨前后端 | 单组件新功能、单 Command 新增 |
| **L** | 大做 | 跨模块/跨前后端，新页面或新集成 | 新页面、新 API 对接、前后端联动 |
| **XL** | 巨工 | 新项目或大型重构，多阶段多 Agent | 新项目初始化、架构重构 |

**判定规则**：拿不准时**向上取一级**。

## 各级别对应流程

| 环节 | S | M | L | XL |
|------|---|---|---|-----|
| 访谈 | 不走 | 精简（1a+1c+第3轮） | 三轮完整 | 三轮 + 蓝图 |
| Plan 模板 | 3 章 | 6 章 | 9 章 | 9 章 |
| Agent 数量 | 1 实现 | 1-2 实现 | 全团队 | 全团队+多阶段 |
| @debugger | — | 退回时介入 | 必须 | 必须 |
| @cross-checker | — | — | 必须 | 必须 |
| @reviewer | 必须 | 必须 | 必须 | 必须 |
| @e2e-verifier | — | 必须 | 必须 | 必须 |

## 分级铁律

- **TG-01**：@reviewer 所有级别不可省。M+ @e2e-verifier 不可省。L+ @debugger 不可省
- **TG-02**：级别判定后，验证 Agent 不可省。降级须用户确认
- **TG-03**：退回 ≥2 次（M 级）→ 强制引入 @debugger
- **TG-04**：级别判定错误时必须升级并补充验证 Agent
